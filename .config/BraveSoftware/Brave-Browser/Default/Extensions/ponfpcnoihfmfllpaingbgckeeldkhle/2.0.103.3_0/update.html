Bug fixes and improvements

Hello, this is a quick update to inform you about important changes. Don't worry I don't display this kind of message for each update...

What has been fixed

  • YouTube's “Autoplay” automatically turned on if “Disable autoplay” was not checked in “Options > Video player > Autoplay”
  • YouTube's “Theater mode” automatically reset if “Automatically enable YouTube's “Theater mode”” was not checked in “Options > Appearance”
  • YouTube's “Dark theme” automatically reset to “Light theme” if it was not selected in “Options > Theme”
  • Tooltip and volume/speed info bar visible when resizing the video player or toggling the full screen mode

I will fix the other issues as soon as possible.

What's new

  • The “Options” control is back... But you can remove it in “Options > Video player > Controls” if you don't need it
  • You can make a right click on the “Video filters” control to toggle the visibility of the video filters panel
  • The size of the expanded player is different if you choose to control the volume with the mouse wheel “only when the right mouse button is pressed” or not because in some circumstances it was not possible to scroll down

As you have just seen, I have also added a new button to display the controls only when you hover over it with the mouse cursor, and hide them when you move the mouse cursor away from it or from the controls. You can also click it to keep controls visible, then click another time to hide them. This button has been added for people who don't have a large monitor, but if it's not your case and you want controls to always remain visible you can use the following code as custom script in “Options > Custom script”. You can either check the option to execute it automatically when YouTube is loaded in a tab, or execute it only when needed by adding the “Custom script” control and clicking it when you want.

document.documentElement.classList.toggle('efyt-controls-visible');

In the coming weeks I will add an option to place controls within a toolbar fixed at the bottom of the window like in previous version of the extension.

Loop mode

I thought it was obvious but apparently not everybody knows how to use numerical fields so here is a tip. When you want to quickly adjust the loop range, all you have to do is to click a field to give it the focus and use the mouse wheel to adjust the number. You can also maintain the mouse button pressed on the tiny “▲” or “▼” buttons.

Bonus

If you are using one of the built-in YouTube-DeepDark themes you can use the following code as custom theme in “Options > Theme > Custom theme” to style the scrollbar.

html {
  scrollbar-color: var(--main-color) rgba(0, 0, 0, 0);
  scrollbar-width: thin;
}
html:not([hide-scrollbar]) ::-webkit-scrollbar-thumb {
  background: var(--main-color);
}
html:not([hide-scrollbar]) ::-webkit-scrollbar {
  width: 8px;
}
::-webkit-scrollbar-track {
  background: rgba(0, 0, 0, 0);
}
::-webkit-scrollbar-thumb {
  background: var(--main-color);
}
